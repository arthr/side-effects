# Registro de Observações e Decisões de Design

## Diretrizes
*   Sempre que uma decisão de design significativa for tomada (ex: escolha de uma biblioteca, definição de uma estrutura de dados crítica para o jogo, ou resolução de um conflito de regras), registre a decisão, o porquê e a data no arquivo `.logs/design_decisions.md`.
*   Trate este log como parte do contexto do projeto que deve ser lido antes de grandes alterações.
*   Em cada nova interação (comando), o agente deve fazer um "check-in" de contexto, lendo este arquivo de log e o `tech-stack.md` para garantir consistência.
